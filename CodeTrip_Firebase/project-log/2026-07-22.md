## Gemini API key 트러블슈팅 및 알림 오류 처리 보정

### 작업 배경

Gemini 기능 로컬 테스트를 재개하면서 API key를 새로 설정했음에도 동일한 오류 메시지가 반복되는 문제가 있었습니다.
또한 메인 홈 진입 시 공통 Header에서 알림 조회 실패 토스트가 표시되는 문제가 함께 확인되었습니다.

오늘 작업은 다음 두 가지를 중심으로 진행했습니다.

```text
Gemini API key, 프로젝트, 모델명, quota, 서버 상태를 구분하여 원인 추적
공통 Header의 알림 조회 실패 토스트가 불필요하게 노출되지 않도록 보정
```

### 알림 조회 실패 토스트 보정

수정 파일:

```text
src/components/Layout/Header.jsx
```

문제:

```text
메인 홈 진입 시 "알림을 불러오는 데 실패했습니다." 토스트가 표시됨
실제로는 메인 홈 전용 기능이 아니라 공통 Header에서 알림 목록을 조회하는 과정에서 발생
localStorage의 trip_user와 Firebase Auth currentUser 복구 시점이 잠깐 어긋나거나
Realtime Database Rules의 auth.uid 조건과 맞지 않는 순간 permission/auth 계열 오류가 발생할 수 있음
```

수정 내용:

```text
알림 조회 실패 시 error.code와 error.message를 확인
permission, auth, login, 로그인 계열 오류는 인증 경계 상황으로 판단
이 경우 빨간 토스트를 띄우지 않고 notifications와 unreadCount만 빈 상태로 초기화
그 외 예상하지 못한 오류일 때만 기존 "알림을 불러오는 데 실패했습니다." 토스트 유지
```

결과:

```text
로그인 복구 타이밍 또는 권한 경계 상황으로 인한 불필요한 오류 토스트 노출 감소
실제 알림 데이터가 없는 경우에는 빈 알림 목록으로 정상 처리
```

### Gemini API key 트러블슈팅

확인한 문제 흐름:

```text
1. Gemini API key를 새로 입력하고 Vite 서버를 재실행했지만 동일한 오류 메시지 반복
2. Google Cloud 사용자 인증 정보 화면에 여러 Gemini 관련 key가 존재함
3. CodeTrip 프로젝트(newagent-9c2a8)에서 만든 key와 별도 Gemini API 프로젝트 key를 구분해야 했음
4. 일부 Gemini API key는 서비스 계정에 바인딩되어 웹사이트 HTTP referrer 제한을 걸 수 없었음
5. 웹사이트 제한은 일단 후순위로 두고 API 제한과 서비스 계정 바인딩 상태를 확인
6. .env의 VITE_GEMINI_MODEL이 종료된 gemini-2.0-flash로 남아 있음을 확인
7. 모델명을 gemini-3.5-flash로 변경한 뒤 API key 인증 단계는 통과
8. 이후 503 Service Unavailable 응답이 발생하여 Gemini 모델 서버 혼잡 상태로 판단
```

상태 코드별 판단 기준:

```text
401 Unauthorized
- API key 값, 프로젝트, API 사용 설정, 서비스 계정 바인딩, API 제한 설정 문제 가능성이 큼
- CodeTrip 프로젝트에서 생성한 Gemini API key인지 확인 필요

429 RESOURCE_EXHAUSTED
- quota, 사용량 한도, 결제/크레딧 문제 가능성이 큼
- Google AI Studio 또는 Google Cloud Billing에서 quota/credit 상태 확인 필요

503 Service Unavailable
- API key 인증 이후 Gemini 모델 서버가 일시적으로 혼잡한 상태
- 코드나 key 설정 문제보다는 모델 수요 과부하로 판단
- 잠시 후 재시도하거나 Lite 계열 모델 검토
```


변경 이유:

```text
gemini-2.0-flash는 현재 사용 가능한 모델 상태와 맞지 않아 호출 실패 원인이 될 수 있음
API key를 변경해도 모델명이 오래된 상태면 동일한 실패가 반복될 수 있음
README와 Gemini 프롬프트 설계 문서의 환경변수 예시도 코드와 동일하게 변경
```
