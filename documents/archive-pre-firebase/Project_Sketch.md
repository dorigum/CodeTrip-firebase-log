## 2. 프로젝트 최종 방향

### 컨셉: "코드를 짜듯 꼼꼼하게 설계된 여행 가이드"

> **CodeTrip** — 코드 주석처럼 세밀한 여행 가이드를 제공하는 서비스

- 여행지 상세 정보를 코드 주석(`// TIP:`, `/* 주의사항 */`) 스타일의 UI로 표현
- 날씨 API를 연동하여 현재 날씨에 맞는 여행지를 추천하는 기능

### 페르소나

| 항목 | 내용 |
|------|------|
| 이름 | 정해진 (26세, 취준생 / 주니어 개발자) |
| 특징 | 어디 갈지 고르는 선택 장애, 날씨 좋으면 일단 나가고 싶은 타입 |
| 니즈 | 지금 내 상황(날씨/위치/시간)에 맞는 딱 한 곳 추천받기 |

## 3. 핵심 기능 아이디어

### 3.1 지역별/테마별 여행지 소개
- 여행지 상세 페이지 (게시판 + 댓글 형식)
- 설명은 코드 주석 스타일 UI로 표현
- Kakao Maps API로 위치 마커 표시
- 지역별(제주/부산/서울), 카테고리별(맛집/명소/숙소) 다중 필터

### 3.2 나의 여행 리스트 (즐겨찾기)
- LocalStorage로 새로고침 후에도 목록 유지
- 장바구니 형식의 UI

### 3.3 날씨 기반 여행지 랜덤 추첨
- 날씨 API로 현재 기상 상태 파악
- 날씨에 따른 필터링 로직:
  - 맑음 ☀️ → 야외 명소 / 인생샷 명소
  - 비/눈 🌧️ → 실내 전시회 / 문화 공간
  - 구름 ☁️ → 산책로 / 로컬 행사
- 슬롯머신 스타일 랜덤 뽑기 UI (Framer Motion 활용 예정)

### 3.4 축제/행사 정보 (보류)
- 기간별 지역 축제 안내 + 지도

### 추천 기능 아이디어**

1. **즉흥 여행 추천 버튼** ✔
    
    사용자가 길게 검색하지 않아도 오늘 떠나기, 이번 주말 떠나기, 반차 쓰고 가기, 1박 2일, 당일치기 같은 조건만 누르면 여행지를 추천해주는 기능.
    
    예시 조건:
    - 출발 지역
    - 이동 가능 시간
    - 예산
    - 여행 스타일: 힐링 / 맛집 / 자연 / 바다 / 전시 / 혼자 / 친구 / 커플
    - 지금 기분: 답답함 / 조용히 쉬고 싶음 / 신나는 곳 / 아무 생각 안 하고 싶음
      
2. **“선택 장애” 사용자를 위한 랜덤 큐레이션**
    
    완전 랜덤이 아니라 사용자의 취향을 반영한 랜덤 추천이면 좋아.
    
    예시:
    - “내 취향 기반 랜덤 여행지 뽑기”
    - “3개만 추천받기”
    - “싫으면 다시 뽑기”
    - “오늘의 즉흥 여행 카드”
    
    여행지를 카드처럼 보여주고, 사용자가 좋아요 / 별로예요 / 나중에를 누르면 취향 데이터가 쌓이게 만들 수 있어.
    
3. **관심 지역 기반 알림**
    
    네가 말한 기능은 방향이 좋아. 단순히 “행사 알림”만 주기보다, 즉흥 여행자에게 맞게 **갈 만한 타이밍**을 알려주는 식이면 더 차별화돼.
    
    예시:
    - “이번 주말 강릉에서 야시장 열려요”
    - “부산 숙소 가격이 평소보다 낮아요”
    - “전주 당일치기 코스로 가기 좋은 날씨예요”
    - “관심 지역에 봄꽃 축제가 시작됐어요”
      
4. **날씨 + 행사 + 교통 기반 추천**
    
    즉흥 여행자는 “지금 갈 수 있나?”가 중요하니까 여행지 자체보다 **실행 가능성**을 보여주면 좋아.
    
    예시:
    - 날씨 좋음
    - 기차/버스 이동 가능
    - 당일치기 가능
    - 행사 있음
    - 숙소 여유 있음
    - 예상 비용 적당함
    
    추천 문구도 이렇게 줄 수 있어:
    
    > “이번 주말엔 대전보다 군산이 더 좋아요. 날씨가 맑고, 이동 시간이 짧고, 근처 행사도 있어요.”
    
5. **사용자 취향 프로필** ✔
    
    회원가입 후 관심 지역만 받기보다, 짧은 취향 설문을 추가하면 맞춤성이 훨씬 좋아져.
    
    예시 질문:
    - 혼자 여행을 선호하나요?
    - 이동 시간이 긴 여행도 괜찮나요?
    - 자연/도시/맛집/문화 중 선호하는 것은?
    - 예산은 어느 정도가 편한가요?
    - 계획적인 여행 vs 즉흥적인 여행
    - 사람이 많은 곳도 괜찮은지
    
    이걸 바탕으로 힐링형, 맛집탐험형, 전시산책형, 바다충전형, 무계획즉흥형 같은 여행 성향을 만들어도 좋아.
    
6. **“지금 내 상황” 기반 추천** ✔
    
    다른 여행 정보 사이트와 가장 다르게 만들 수 있는 부분이야.
    
    예시:
    - “퇴근 후 갈 수 있는 곳”
    - “내일 아침 출발 가능한 곳”
    - “혼자 가도 어색하지 않은 곳”
    - “돈 많이 안 쓰고 기분전환 가능한 곳”
    - “사람 적은 조용한 여행지”
    - “갑자기 떠나도 준비물 적은 곳”
      
7. **여행 패키지처럼 묶어서 추천**
    
    여행지만 추천하면 사용자가 다시 숙소, 교통, 코스를 찾아야 하니까 피로도가 남아.  
    그래서 “추천 여행지 + 간단 일정 + 교통 + 예상 비용”을 한 번에 보여주면 좋아.
    
    예시:
    - 여행지: 속초
    - 추천 이유: 바다, 카페, 당일/1박 가능
    - 이동: 서울 기준 버스 약 2시간 30분
    - 예상 비용: 8만 원부터
    - 추천 코스: 중앙시장 → 영금정 → 해변 카페
    - 어울리는 사용자: 혼자 바람 쐬고 싶은 사람
      
8. **알림을 더 개인화하기**
    
    알림도 단순한 푸시보다 사용자의 상황에 맞게 보내면 좋아.
    
    예시:
    - “이번 주말 비 안 오는 가까운 여행지 3곳”
    - “관심 지역 제주에 항공권 특가가 떴어요”
    - “당신이 저장한 전주와 비슷한 감성의 여행지가 있어요”
    - “최근 힐링 여행지를 많이 봤다면, 이번엔 담양 어때요?”
      
9. **여행 후 피드백 기반 추천 개선**
    
    사용자가 여행지를 다녀온 뒤 짧게 평가하게 만들면 추천 품질이 좋아져.
    
    예시:
    - 실제로 다녀왔나요?
    - 만족했나요?
    - 사람이 많았나요?
    - 다시 가고 싶나요?
    - 즉흥 여행으로 적합했나요?
    
    이 데이터는 이후 추천 알고리즘에 활용할 수 있어.
    

**개인적으로 가장 추천하는 차별화 방향**

서비스의 핵심 문구를 이렇게 잡으면 좋아 보여:

> “계획이 없어도 떠날 수 있게, 지금 나에게 맞는 여행지를 골라주는 서비스”

기능 우선순위는 이렇게 가면 좋을 것 같아.

1. **관심 지역 및 여행 취향 설정**
2. **즉흥 여행 추천 버튼**
3. **날씨/행사/이동시간 기반 추천**
4. **관심 지역 알림**
5. **좋아요/싫어요 기반 개인화 추천**

이렇게 만들면 단순 여행 정보 사이트가 아니라, **결정이 귀찮거나 힘든 사람을 대신해서 여행 결정을 가볍게 만들어주는 서비스**가 돼. 특히 네가 프로젝트를 시작한 개인적인 경험이랑도 잘 맞아서, 기획 의도가 기능에 자연스럽게 살아날 것 같아.

### ==바로 구현하기 좋은 기능==

1. **관심 지역 설정** ✔
    가능해.  
    현재 users 테이블에는 관심 지역 필드가 없어서 user_preferences 같은 테이블을 새로 만들면 돼.
    
    예:
    - 관심 지역: 서울, 강원, 부산 등
    - 선호 테마: 자연, 문화시설, 축제, 바다, 맛집 등
    - 여행 스타일: 당일치기, 1박 2일, 혼자 여행, 조용한 여행
    
    설정 위치는 기존 Settings.jsx에 “여행 취향 설정” 섹션을 추가하면 자연스러워.
    
2. **관심 지역 기반 축제/행사 추천** ✔
    가능해.
    이미 /travel/festivals API가 있고, 서버 캐시도 있어서 사용자의 관심 지역과 축제 주소를 매칭해서 보여줄 수 있어.
    
    다만 “푸시 알림”까지 하려면 단계가 나뉘어:
    
    - 앱 안에서 보여주는 알림: 바로 가능
    - 로그인 시 “관심 지역 새 행사” 배너: 가능
    - 이메일/브라우저 푸시 알림: 추가 인프라 필요
      
3. **사용자 맞춤 베스트 여행지 추천** ✔
    가능해.  
    현재 /travel/near, /travel/random, /travel API가 있으니까 여기에 사용자 취향 조건을 섞으면 돼.
    
    추천 점수 예:
    - 관심 지역 일치
    - 선호 테마 일치
    - 위시리스트에 저장한 여행지와 유사
    - 현재 날씨와 어울림
    - 이미지가 있는 여행지 우선
    - 축제 기간과 겹치는 지역 우선
      
4. **즉흥 여행 추천 버튼**
    아주 잘 맞아.  
    이미 홈에 슬롯머신 추천이 있어서 이걸 “즉흥 여행 추천”으로 발전시키면 돼.
    
    예:
    - 오늘 떠나기
    - 이번 주말
    - 당일치기
    - 1박 2일
    - 혼자 조용히
    - 기분전환
    
    지금 구조에서는 UI와 필터 로직만 추가해도 MVP는 만들 수 있어.
    
5. **좋아요/싫어요 기반 추천 개선** ✔
    가능해.  
    새 테이블 하나만 추가하면 돼.
    
    예:
    - user_travel_feedback
    - user_id
    - content_id
    - reaction: like / dislike / skip
    
    이 데이터가 쌓이면 랜덤 추천이 “그냥 랜덤”이 아니라 “내 취향 기반 랜덤”이 돼.




**추천 구현 범위별 일정**

가장 현실적인 **1차 MVP**는 이 정도야:

1. 사용자가 관심 지역과 여행 스타일을 설정한다.
2. 홈에서 “내 취향 기반 즉흥 여행지”를 추천한다.
3. 관심 지역의 축제/행사를 보여준다.
4. 추천 결과에 좋아요/별로예요를 누를 수 있다.


조금 더 발표용/포트폴리오용으로 완성도를 올리면:

1. 마이페이지 또는 설정 페이지에 취향 관리 UI 추가
2. 추천 이유 표시
3. 관심 지역 행사 알림 섹션
4. 피드백 기반 추천 개선
5. 빈 상태/로딩/비로그인 사용자 처리
6. 반응형 UI 정리

## 추천 맞춤 기능 (우선순위 순)

### 1. 관심지역 → 홈 추천 연동 ⭐ (쉬운 구현, 높은 임팩트)

**현재 상태**: Settings에서 관심지역 최대 3개를 저장하는 기능이 완전히 구현돼 있고(`user_favorite_regions` 테이블, `getFavoriteRegions` API), 저장도 잘 됩니다. 그런데 **Home.jsx가 이 데이터를 전혀 사용하지 않습니다.** GPS 위치만 보고 추천합니다.

**추가할 내용**: 로그인 상태라면 `getFavoriteRegions()`를 호출해서, "내 관심지역 추천" 섹션을 홈에 별도로 보여주는 것.

```
홈 화면 구성 (로그인 시)
├── [기존] 현재 위치 기반 추천
└── [추가] 관심지역 기반 추천 (서울, 제주 등 저장된 지역 카드)
```

---

### 2. 최근 본 여행지 (Recently Viewed) ⭐ (프론트만 수정)

**현재 상태**: TravelDetail에 진입해도 어디에도 기록이 남지 않습니다.

**추가할 내용**: `TravelDetail.jsx` 진입 시 `localStorage`에 방문 기록을 저장하고, MyPage 또는 홈 하단에 "최근 본 여행지" 가로 스크롤 카드로 표시.

- 백엔드 불필요, localStorage + Zustand persist로 구현 가능
- 최대 10개 유지 (FIFO 방식)

---

### 3. 탐색 페이지 — 관심지역 자동 선택 (쉬운 구현)

**현재 상태**: Explore 진입 시 항상 필터가 비어있는 상태로 시작합니다.

**추가할 내용**: 로그인 사용자의 저장된 관심지역을 `useExploreStore`의 `selectedRegions` 초기값으로 자동 세팅. 단, 사용자가 직접 변경하면 무시.

---

### 4. 위시리스트 통계 위젯 (MyPage 개선)

**현재 상태**: MyPage는 위시리스트 목록만 보여줍니다.

**추가할 내용**: 이미 있는 `wishlistItems` 데이터로 계산 가능한 통계를 상단에 표시.

```
📊 내 여행 통계
├── 총 저장 여행지: 24곳
├── 폴더 수: 3개
├── 가장 많이 담은 지역: 제주 (8곳)
└── 가장 많이 담은 테마: 관광지 (15곳)
```

---

### 5. 최근 검색어 (헤더 검색창)

**현재 상태**: 검색창에 아무 기억이 없습니다.

**추가할 내용**: 검색어 제출 시 localStorage에 최근 5개 저장, 검색창 클릭 시 드롭다운으로 표시.
