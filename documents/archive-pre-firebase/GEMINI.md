## Gemini Added Memories
- 앞으로 커밋 메시지를 작성할 때 항상 'YYMMDD 커밋 메시지' 형식을 사용한다. (예: 260330 ex0327 과제(1, 2번) 코드 작성)
- 사용자의 요청에 따라 모든 답변은 한국어로 작성한다.
- 사용자가 명시적으로 지시하기 전까지는 절대로 커밋(commit)이나 푸시(push)를 진행하지 않는다.
- 모든 문서를 작성할 때 내용이 중복되지 않는 한, 절대 내용을 마음대로 삭제하거나 수정하지 않고 이어서 작성한다. (기존 내용 보존 원칙)
- 이후 수정되거나 추가되는 사항은 넘버링(1. 2. 3.) 하여 내용을 구분한다.
- 모든 작업 전에 gemini.md 에 적힌 규칙들을 먼저 확인한 후 작업을 실행한다.
