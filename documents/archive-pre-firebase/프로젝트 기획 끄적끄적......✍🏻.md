## ==페르소나/프로젝트 테마 기획==
#### 1. 테마: "오늘의 날씨가 정해주는 P(즉흥형)들의 여행"
- **페르소나:** 계획 세우는 건 귀찮지만, 날씨 좋은 날 집에 있기는 싫은 2030 직장인 '이즉흥' 씨.

- **주요 이용 타겟:** 날씨와 기분에 따라 목적지를 정하는 즉흥 여행객.

- **특화 기능:**
    - **날씨 맞춤 추천:** 날씨 API를 연동해 '맑음'이면 사진 잘 나오는 야외 명소나 벚꽃길을, '비'가 오면 실내 전시회나 아늑한 북카페를 우선 추천
    - **지도 기반 '현재 위치' 주변 추천:** "내 주변 5km 이내의 강아지 산책로" 같은 위치 기반 인터랙션 강조

- **프론트엔드 포인트:** 날씨 아이콘 애니메이션, 날씨 데이터에 따른 동적 배경 변경(비 오는 날은 웹사이트 배경에도 비가 내리는 효과 등)


#### 2. 테마: "멍뭉이와 함께하는 사계절 맵" (반려동물 타겟)
- **페르소나:** 혼자 여행하는 것보다 강아지 '초코'와의 추억이 더 소중한 반려인 '김집사' 씨.

- **주요 이용 타겟:** 반려동물 동반 가능 여부가 여행지 선정의 1순위인 1,500만 반려가구.

- **특화 기능:**
    - **반려동물 전용 필터:** 벚꽃 명소 중에서도 '대형견 가능', '리드줄 필수', '배변봉투 수거함 위치' 등을 JSON 데이터로 상세히 구분.
    - **지도 마커 차별화:** 일반 관광지는 파란 마커, 반려동물 환영 장소는 발바닥 마커로 표시.

- **프론트엔드 포인트:** 복잡한 다중 카테고리 필터링 로직, 반려동물 동반 에티켓(주석 스타일) 카드 UI

💡 한국관광공사 API에서도 최근 반려동물 동반 여행 정보를 강화하고 있어서 데이터를 가져오기 좋습니다.


#### 3. 테마: "MZ세대의 계절 한정 숏트립" (시즌 & 갓생 테마)
- **페르소나:** 남들 다 가는 시기를 놓치기 싫고, 제철 풍경을 배경으로 인생샷을 남기고 싶은 '트렌드' 씨.
    
- **주요 이용 타겟:** 벚꽃, 단풍, 은하수 등 특정 시기에만 볼 수 있는 풍경을 찾는 20대 대학생 및 사회초년생.
    
- **특화 기능:**
    - **시즌 한정 큐레이션:** 현재 날짜를 계산해 '지금 가장 핫한 벚꽃 실시간 개화 상황'(API 연동 또는 가상 데이터) 제공.
    - **테마별 경로 추천:** '강아지와 함께 걷는 벚꽃길 30분 코스'처럼 지도 위에 이동 경로를 그려주는 기능
    
- **프론트엔드 포인트:** `react-router-dom`을 활용한 매끄러운 페이지 전환, 가고 싶은 곳을 저장하는 **관심 목록(LocalStorage)** 관리.


🎲 **1번 테마 확장 ver.**
#### 🗺️ 프로젝트 테마: "오늘의 날씨가 정해주는 운명적 여행, [오늘어때 (가제)]"

- **페르소나 (Target)**
	- **이름:** 정해진(26세, 취준생 혹은 주니어 개발자)
    
- **특징:**
    - 여행은 가고 싶지만, 어디를 갈지 고르는 '선택 장애'가 있음.
    - 날씨가 좋으면 일단 나가고 싶어 하지만, 비가 오면 실내를 선호하는 환경 민감형.
    - 복잡한 계획보다는 **"지금 내 상황(날씨/위치/시간)에 맞는 딱 한 곳"** 을 추천받길 원함.

#### 2. 핵심 기능: "날씨 기반의 명분 있는 랜덤 추천"
단순히 버튼을 누르면 나오는 게 아니라, 사용자의 환경 데이터를 활용해 **'명분'** 을 만들어줍니다.

- **Step 1: 환경 데이터 수집:** 날씨 API를 통해 현재 위치의 기상 상태(온도, 하늘 상태, 강수 유무)를 파악합니다.
   
- **Step 2: 필터링 알고리즘 (명분 생성):**
    - **맑음 ☀️:** "하늘이 이렇게 예쁜데 집에 있을 거예요? 인생샷 보장하는 **야외 명소** 추천!"
    - **비/눈 🌧️:** "비 오는 날의 운치를 즐겨보세요. 포근한 **실내 전시회나 문화 공간** 추천!"
    - **구름 많음 ☁️:** "걷기 딱 좋은 온도네요! 부담 없는 **동네 산책로와 로컬 행사** 추천!"

- **Step 3: 슬롯머신/룰렛 UI:** 위 조건에 필터링된 JSON 데이터 중 하나를 랜덤으로 뽑아 슬롯머신처럼 화려하게 보여줍니다. (React의 `Framer Motion`을 쓰기 딱 좋은 구간이죠!)

#### 🛠️ 프론트엔드 구현 포인트

##### 1. 카카오맵 API와의 시너지
- **랜덤 추천지 마커 표시:** 뽑힌 여행지를 지도 중앙에 딱 띄워주고, 주변의 '테마별(강, 산, 바다)' 명소를 함께 노출합니다.
- **로드뷰 기능:** 카카오맵의 로드뷰 API를 활용해, 뽑힌 장소를 가기 전 미리 360도로 확인하게 해주면 완성도가 확 올라갑니다.

##### 2. "주석 스타일"의 가이드 UI
- 여행지 상세 페이지에 **[에디터 도연의 꼼꼼 가이드]** 라는 섹션을 만드세요.
    
- 코드 주석처럼 `// TIP: 여기는 오후 4시 노을이 제일 예뻐요` 혹은 `/* 주의사항: 월요일은 휴무입니다 */` 같은 디자인 요소를 넣으면, 도연님만의 고유한 브랜드가 녹아든 포트폴리오가 됩니다.

##### 3. 날씨 테마 동적 UI
- 날씨 API 상태 값에 따라 웹사이트의 배경색(Background Color)이나 아이콘이 변하게 만드세요. (맑으면 하늘색, 비 오면 차분한 그레이톤)

💡 정리하자면...
도연님은 **"랜덤이라는 재미"** 에 **"날씨라는 합리적인 이유"** 를 더한 서비스를 만드는 거예요. 백엔드가 없어도 사용자의 브라우저 정보(날씨, 위치)를 실시간으로 받아와서 처리하기 때문에 충분히 기술적인 도전 과제도 많습니다.

이 컨셉대로라면 **지역별/테마별** 카테고리도 자연스럽게 녹일 수 있어요. (예: 맑은 날 + 바다 테마만 모아서 랜덤 돌리기)

## ==프로젝트 컨셉 정하기==
#### 💡 주제 선정 시 팁 (Cheatsheet)

| **체크리스트**   | **설명**                                                |
| ----------- | ----------------------------------------------------- |
| **API 가용성** | 공공데이터 포털이나 무료 API(TMDB, Spotify 등)를 쓸 수 있는지 확인하세요.    |
| **기술적 도전**  | 단순 CRUD보다는 **성능 최적화, 실시간 통신, 복잡한 상태 관리** 중 하나는 포함하세요. |
| **스토리텔링**   | "내가 이걸 왜 만들었는지"에 대한 명확한 이유가 있어야 포트폴리오에서 빛을 발합니다.      |

#### 1. "로컬 페스티벌 트래커" (시즌 & 타이밍 중심)

단순한 장소 소개가 아니라, 지금 당장 혹은 이번 주말에 열리는 **'행사'와 '축제'** 에 초점을 맞춘 컨셉입니다.

- **차별화 포인트:** 정적인 여행지 정보보다 "언제 가야 하는가"라는 동적인 정보를 다룹니다.
    
- **핵심 기능:**
    - **D-Day 카운트다운:** 축제 시작일까지 남은 시간을 시각적으로 표시.
    - **월별/계절별 캘린더 뷰:** React Calendar 라이브러리를 활용해 월별 축제 일정을 한눈에 확인.
    - **지역별 축제 지도:** 지도 위에 현재 진행 중인 축제만 특별한 마커(예: 폭죽 아이콘)로 표시.
    
- **기술적 재미:** 날짜 데이터 계산 로직, 캘린더 라이브러리 커스텀


#### 2. "취향 맞춤 '갓생' 로컬 투어" (활동 & 테마 중심)

**'경험'과 '활동'** 에 집중하는 컨셉입니다. (예: 원데이 클래스, 플로깅, 출사 명소, 러닝 코스)

- **차별화 포인트:** "어떻게 시간을 보낼까"를 해결해 줍니다.
    
- **핵심 기능:**
    - **취향 필터링:** #사진찍기좋은 #활동적인 #조용한 #아이와함께 등 세밀한 태그 검색.
    - **나만의 루트 만들기:** JSON 데이터를 활용해 가고 싶은 장소 3~4곳을 선택하면 지도 위에 선으로 연결해주는 기능.
    - **장소별 '혼잡도' 체크:** (가상 데이터를 활용해) 시간대별 방문객 그래프 시각화.
    
- **기술적 재미:** 복잡한 다중 필터링 로직, 드래그 앤 드롭으로 루트 순서 바꾸기 (`react-beautiful-dnd`)


#### 3. "디지털 노마드를 위한 '워크앤스테이' 가이드" (타겟 중심)

최근 트렌드인 **'워케이션(Work+Vacation)'** 명소를 소개하는 컨셉입니다.

- **차별화 포인트:** 일반 관광객이 아닌, 일하며 여행하는 사람들을 위한 실무적인 정보 제공.
    
- **핵심 기능:**
    - **작업 환경 정보:** Wi-Fi 속도, 콘센트 유무, 조용한 정도 등을 아이콘으로 직관적으로 표시.
    - **숙소 + 공유 오피스 결합 정보:** 인근에 일하기 좋은 카페나 도서관 정보를 함께 매칭.
    - **리뷰 주석 시스템:** 도연님이 칭찬받으셨던 '꼼꼼한 주석' 역량을 살려, 각 장소에 대한 아주 세세한 '에디터의 팁' 섹션 운영.
    
- **기술적 재미:** 상세한 인포그래픽 구현, 로컬 스토리지 기반의 '최근 본 워케이션 장소' 저장.


#### 4. 💡 프로젝트 시나리오 제안: "나만의 숨은 명소 아카이브"

이 프로젝트도 **'상세한 가이드'** 컨셉으로 가면 좋을 것 같아요.

1. **메인:** 예쁜 카드 레이아웃으로 여행지 나열
2. **검색:** "제주도 카페"라고 치면 해당 조건에 맞는 카드만 남고 나머지는 부드럽게 사라지는 애니메이션 (`Framer Motion` 활용).
3. **디테일:** 해당 장소에 대한 도연님만의 팁(주석처럼 세세한 설명)을 예쁜 UI로 배치

## ==활용 기술 범위(참고)==

**✔ 주제 선정하기 전에 정리한 내용이라서 추천 주제가 안 맞을 수 있음❗❗❗❗❗❗**

#### 1. Browser Storage 활용 (로컬 중심)
별도의 서버 없이 브라우저의 `LocalStorage`나 `IndexedDB`를 데이터베이스처럼 사용하는 방식입니다.

- **추천 주제:** 개인화 대시보드, 투두 리스트(고급 버전), 가계부, 루틴 트래커
    
- **구현해볼 만한 기능:**
    - **CRUD:** 일상적인 데이터 저장, 수정, 삭제.
    - **다크모드/테마 설정:** 유저의 디자인 선호도를 저장해 재방문 시 유지.
    - **데이터 시각화:** 저장된 데이터를 기반으로 `Recharts`나 `Chart.js`를 이용해 통계 그래프 구현
    - **JSON 파일 내보내기/불러오기:** 내 데이터를 백업하고 복구하는 기능

#### 2. Public API 활용 (데이터 중심)
이미 잘 만들어진 외부 데이터를 가져와서 나만의 방식으로 가공해 보여주는 방식입니다.

- **추천 주제:** 영화 정보 큐레이션(TMDB), 실시간 날씨 기반 코디 추천(OpenWeather), 기술 블로그 모아보기.
    
- **구현해볼 만한 기능:**
    - **비동기 처리:** `TanStack Query(React Query)`를 사용해 데이터 로딩, 에러 핸들링, 캐싱 처리.
    - **검색 및 필터링:** 수천 개의 API 데이터 중 사용자가 원하는 것만 필터링.
    - **무한 스크롤:** 다음 페이지의 데이터를 계속 불러오는 UX 구현.

#### 3. BaaS (Backend as a Service) 활용 (실무형)
**Firebase**나 **Supabase** 같은 서비스를 사용하면, 백엔드 코드를 한 줄도 안 짜고도 실제 DB와 회원가입 기능을 쓸 수 있습니다. **취업용 포트폴리오라면 이 방식을 가장 추천합니다.**

- **추천 주제:** 실시간 채팅방, 커뮤니티 게시판, 소셜 미디어 클론
    
- **구현해볼 만한 기능:**
    - **소셜 로그인:** 구글이나 카카오 계정으로 간편 로그인 구현
    - **이미지 업로드:** 프로필 사진이나 게시물 이미지를 스토리지에 저장
    - **실시간 동기화:** 누군가 댓글을 달면 새로고침 없이 내 화면에도 바로 뜨는 기능

#### 1. 데이터 소스 준비 (JSON)
백엔드 대신 `data.json` 파일을 만들거나, 공공데이터 API를 활용할 수 있습니다.

- **방법 B:** **한국관광공사 국문 관광정보 서비스 API** 활용 (실제 데이터)

#### 2. 핵심 구현 기능 (Scope)
React의 기능을 최대한 활용할 수 있는 요소들이에요.

- **스마트 검색 및 필터링:**
    - 사용자가 입력하는 즉시 결과가 나오는 **실시간 검색**.
    - 지역별(제주, 부산, 서울), 카테고리별(맛집, 명소, 숙소) **다중 필터**.
    
- **상세 페이지 및 동적 라우팅:**
    - `react-router-dom`을 사용하여 각 여행지의 상세 정보를 보여주는 개별 페이지 구현.
    
- **'나의 찜' 리스트 (Wishlist):**
    - **LocalStorage**를 활용해 브라우저를 새로고침해도 내가 즐겨찾기한 여행지가 유지되도록 구현.
    
- **지도 연동 (핵심 포인트):**
    - **Kakao Maps API** 또는 **Google Maps API**를 연동하여 여행지의 위치를 마커로 표시.

#### 3. 기술적 도전 과제 (포트폴리오 어필 포인트)
단순한 조회를 넘어 프론트엔드 역량을 증명할 수 있는 부분입니다.

- **Skeleton UI:** 데이터를 불러오는 동안(API 통신 시) 부드러운 로딩 화면 보여주기.
- **최적화:** 검색어가 입력될 때마다 연산을 하지 않도록 **Debounce** 적용하기.
- **이미지 최적화:** 여행지 사진이 많을 경우 **Lazy Loading**을 적용해 초기 로딩 속도 개선.
- **모바일 퍼스트:** 여행지는 밖에서 폰으로 보는 경우가 많으니 **반응형 디자인**과 **PWA(Progressive Web App)** 도입 고려.

1. **프로젝트의 핵심 기능을 강조한 이름**
>날씨와 랜덤(뽑기)이라는 직관적인 느낌을 담았습니다.

- **날뽑 (날씨가 뽑아준 여행지)**: 입에 착 붙고 기억하기 쉬운 줄임말 스타일입니다.
- **웨더픽 (Weather Pick)**: 날씨가 골라준다는 의미로, 깔끔하고 전문적인 서비스 느낌을 줍니다.
- **오늘의 운명 (Today's Destiny)**: "오늘 네가 갈 곳은 운명적으로 정해져 있어"라는 감성적인 접근입니다.

1. **도연님만의 '주석/가이드' 강점을 살린 이름**
>꼼꼼하게 정보를 기록하고 주석을 다는 도연님의 스타일을 브랜딩에 녹였습니다.

- ✔**코드트립 (Code Trip)**: 코드를 짜듯 꼼꼼하게 설계된 여행 가이드라는 중의적 의미입니다.
- **주석 한 줄 (A Line of Comment)**: 여행지에 대한 세밀한 팁(주석)이 핵심인 서비스라는 정체성을 보여줍니다.
- **가이드로그 (Guide-Log)**: 단순 정보 나열이 아니라 개발자의 시선으로 기록한 여행 일지 느낌입니다.

1. **MZ세대와 즉흥 여행객(P)을 타겟팅한 이름**
>페르소나인 '이즉흥' 씨가 좋아할 만한 트렌디한 느낌입니다.

- 어디갈래 (Where to Go): 가장 본질적인 질문을 이름으로 정해 직관성을 높였습니다.
- ✔**떠날지도 (Map to Leave)**: '지도를 보고 떠난다'와 '떠날지도 모른다'는 중의적 표현으로, 지도 API 활용을 강조합니다.
- 지금여기 (Now & Here): 현재 날씨와 내 위치를 기반으로 한다는 실시간성을 강조합니다.
- ✔**루트 0 (Route 0)**: 아무것도 정해지지 않은 상태(0)에서 시작하는 여행이라는 공학적 느낌의 이름입니다.
