**한 줄 소개**  
CodeTrip은 공공 여행 데이터, 위치/날씨 정보, 사용자 관심 지역, 위시리스트, 커뮤니티를 결합해 사용자가 “지금 나에게 맞는 여행지”를 탐색하고 저장하고 공유할 수 있게 만든 React + Express 기반 여행 큐레이션 서비스입니다.

**프로젝트 아이덴티티**  
핵심 정체성은 Code Vibe Travel Platform입니다. 화면 곳곳에 RUN_FILTER.SH, save_to_folder.sh, node_description.log, system.env 같은 개발자 친화적인 문구와 터미널 스타일 UI가 들어가 있어, 일반적인 여행 앱과 차별화됩니다.

강조하면 좋은 표현은 이렇습니다.

> “CodeTrip은 여행지를 단순히 보여주는 서비스가 아니라, 개발자가 코드를 탐색하듯 여행 데이터를 필터링하고, 저장하고, 기록하고, 공유하는 여행 큐레이션 플랫폼입니다.”

**프론트엔드 분석**  
프론트엔드는 React 19, Vite, TailwindCSS, Zustand, React Router 기반 SPA입니다. 주요 구조는 src/main.jsx, src/App.jsx에 잘 드러납니다.

주요 화면은 Home, Explore, TravelDetail, Festivals, Board, MyPage, MyActivity, Settings로 구성되어 있습니다. 즉, 단순 목록형 앱이 아니라 **탐색 → 상세 확인 → 저장 → 일정/메모 관리 → 커뮤니티 공유 → 활동 기록 확인**까지 이어지는 사용자 여정이 있습니다.

강조할 프론트엔드 기술력:

- **Zustand 기반 전역 상태 관리**: 위시리스트, 지역 필터, 탐색 상태, 인증 상태를 페이지 간 유지합니다.
- **사용자 경험 중심 탐색 UX**: Explore 페이지는 지역/테마 필터, 정렬, 페이지네이션, 스크롤 복원까지 구현되어 있습니다.
- **개인화 UI**: 로그인 사용자의 관심 지역을 기반으로 MY_REGIONS.SH 필터를 제공하고, 위시리스트 상태도 즉시 반영합니다.
- **카카오맵 연동 상세 페이지**: 여행지 좌표 기반 지도 표시와 카카오맵 길찾기 연결이 가능합니다.
- **Markdown 커뮤니티 에디터**: src/components/MarkdownEditor.jsx는 GFM, 툴바, 편집/미리보기/분할 모드를 지원합니다.
- **브랜드 UI 일관성**: Tailwind 테마 색상, 모노스페이스 문구, 터미널 모달, 개발자형 라벨링으로 서비스 개성이 강합니다.

**백엔드 분석**  
백엔드는 Express + MySQL 기반이며, 라우터가 도메인별로 분리되어 있습니다. 진입점은 server/index.js입니다.

주요 백엔드 도메인:

- 인증: 회원가입, 로그인, JWT 발급, 비밀번호 재설정
- 사용자: 프로필 이미지 업로드, 닉네임 수정, 비밀번호 변경, 관심 지역 설정
- 여행 데이터: 공공 TourAPI 프록시, 캐싱, 필터링, 축제 데이터 제공
- 위시리스트: 여행지 저장, 폴더 관리, 일정, 메모/체크리스트
- 커뮤니티: 게시글, 댓글, 좋아요, 여행지 태그
- 활동 기록: 내가 쓴 글, 댓글, 좋아요, 여행지 댓글 조회
- 알림: 관심 지역의 신규 여행지 알림

가장 강조할 백엔드 포인트는 server/services/travelCache.js와 server/routes/travelRoutes.js입니다. 서버 시작 시 약 6만 건 규모의 여행 데이터를 미리 캐싱하고, 매일 새벽 3시에 갱신하도록 설계되어 있습니다. 발표에서는 “외부 API 의존도를 줄이고 응답 속도와 안정성을 확보한 캐싱 계층”이라고 설명하면 좋습니다.

**사용자 맞춤 기능**  
이 프로젝트의 개인화 기능은 꽤 설득력 있습니다.

- **관심 지역 기반 추천**: 사용자가 최대 3개의 관심 지역을 설정하면, 탐색 화면과 즉흥 추천에 반영됩니다.
- **날씨 기반 추천**: Open-Meteo 날씨 코드를 해석해 맑음/흐림/비/눈/폭풍 등으로 분류하고, 날씨에 어울리는 키워드 기반 여행지를 추천합니다.
- **즉흥 여행 슬롯머신**: 로그인 사용자는 관심 지역 + 현재 날씨 + 여행지 점수화 로직을 바탕으로 랜덤하지만 맥락 있는 여행지를 추천받습니다.
- **여행 폴더 시스템**: 위시리스트를 단순 저장이 아니라 여행별 폴더, 시작일/종료일, 메모, 체크리스트로 관리합니다.
- **내 활동 대시보드**: 내가 쓴 게시글, 댓글, 좋아요, 여행지 댓글을 모아 사용자 활동 히스토리를 제공합니다.
- **관심 지역 신규 여행지 알림**: 캐시 갱신 시 새 여행지가 감지되면 관심 지역 사용자에게 알림을 생성합니다.

**강조하면 좋은 기술력**  
PPT에서 기술 강점으로 잡을 만한 것은 아래 5개입니다.

1. **대용량 공공데이터 캐싱 구조**  
    TourAPI 데이터를 서버 메모리에 캐싱하고 정렬본을 미리 만들어 탐색 응답 속도를 높였습니다.
    
2. **외부 API 안정화 계층**  
    TourAPI 프록시, 2시간 캐시, 429 발생 시 일시 차단 로직을 통해 외부 API 장애와 호출 제한에 대응합니다.
    
3. **개인화 추천 알고리즘**  
    사용자 관심 지역, 현재 날씨, 이미지 보유 여부, 콘텐츠 타입, 지역 일치 여부를 점수화해 추천 품질을 높입니다.
    
4. **커뮤니티와 여행 데이터의 결합**  
    게시글에 여행지 태그를 연결하고, 태그를 통해 다시 여행지 상세 페이지로 이동할 수 있어 정보 탐색과 사용자 콘텐츠가 연결됩니다.
    
5. **운영 배포 고려**  
    docker-compose.yml, nginx.conf를 통해 MySQL, Express, React/Nginx를 컨테이너 기반으로 배포할 수 있게 구성되어 있습니다.

    기존 여행 서비스는 정보 나열 중심이고, 사용자의 현재 상황과 취향 반영이 약하다는 문제 제기

    위치, 날씨, 관심 지역, 저장 기록, 커뮤니티를 결합한 맞춤형 여행 경험
    
4. 프론트엔드 구조  
    React SPA, Zustand 상태 관리, 라우팅, 주요 화면 플로우
    
5. 백엔드 구조  
    Express API, MySQL, JWT 인증, 도메인별 라우터, 데이터 캐싱
    
6. 핵심 기능 1: 여행지 탐색  
    지역/테마/키워드/정렬/페이지네이션 기반 탐색
    
7. 핵심 기능 2: 개인화 추천  
    관심 지역 + 날씨 + 점수화 로직 + 슬롯머신 UX
    
8. 핵심 기능 3: 위시리스트와 여행 계획

9. 핵심 기능 4: 커뮤니티  
    Markdown 게시글, 댓글, 좋아요, 여행지 태그
    

10. 기술적 차별점  
    서버 캐싱, TourAPI 프록시, 429 대응, Docker/Nginx 배포
    
11. 프로젝트 아이덴티티  
    터미널 UI, 코드 감성, 개발자 친화적 인터랙션
    
12. 기대 효과 및 확장 가능성  
    AI 추천, 동행 매칭, 경로 최적화, 지도 기반 일정 생성, 모바일 앱 확장
    

**발표용 결론 문장**

> CodeTrip은 공공 여행 데이터를 단순히 보여주는 데서 끝나지 않고, 사용자의 관심 지역과 현재 날씨, 저장한 여행 계획, 커뮤니티 활동을 연결해 개인화된 여행 경험을 제공하는 플랫폼입니다. 특히 개발자 감성의 UI 아이덴티티와 서버 캐싱 기반 데이터 처리 구조를 통해 기능성과 개성을 동시에 갖춘 프로젝트입니다.
