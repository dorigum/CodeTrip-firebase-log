# 여행지 필터링 관련

### 문제 상황

- Explore 페이지에서 필터링 기능을 구현하였으나, 다중 조건 필터링의 경우 필터링된 데이터들의 순서가 올바르지 않음
- 한 페이지에 너무 많은 데이터가 표시됨

### 원인

- 처음 방식은 각 필터 조건별로 한번씩 데이터를 호출한 다음, 받은 데이터를 정렬하여 화면에 보여줌 
➡️ 지역 3개, 테마 2개를 선택한 경우 3 * 2 * 10 = 60개의 데이터가 한 페이지에 표시됨
- 각 테마별로 10개씩 데이터를 받아와 정렬하여 보여주기 때문에, 조건1의 11번째 데이터가 조건2의 1번째 데이터보다 앞에 와야하는 경우가 있을 수 있지만, 실제로는 올바르게 정렬되지 않음

### 시도 1

- 다중 필터링 조건을 거는 경우, API 호출을 백엔드 서버에서 먼저 하고 정렬한 뒤 프론트엔드로 보내줌 
➡️ 만약 한 페이지 당 10개씩 보여줄 때, 3페이지를 조회하면 각 필터 조건별로 10 * 3 = 30개의 데이터를 받아온 뒤 21 ~ 30번째 데이터를 프론트엔드에 보내줘야 순서가 보장됨 
➡️ 하지만 불필요한 데이터도 너무 많이 받아와 낭비가 심함

### 시도 2

- 여행지 API의 데이터가 자주 변하는 데이터는 아니기 때문에, 서버를 시작할 때 모든 데이터를 받아와 메모리에 캐싱 
➡️ API는 이미 정렬된 데이터를 보내주기 때문에 `filter()`로 필터링을 적용해도 데이터는 정렬된 상태 
➡️ 다음과 같은 로직을 통해 매일 오전 3시마다 캐싱된 데이터를 업데이트

```jsx
const msUntil3am = next3am - now; // 다음 오전3시와 현재 시각 사이의 시간을 ms로 계산
setTimeout(() => {
    refreshTravelCache(pool);
    
    // 이후 24시간마다 데이터 업데이트
    setInterval(() => refreshTravelCache(pool), 24 * 60 * 60 * 1000); 
}, msUntil3am); // 오전 3시에 위 블록이 실행됨
```

### 결과

- 여행지 API는 하루에 천 번만 호출이 가능하기 때문에, 호출 횟수가 많아져 조회가 불가능해지는 상황을 예방
- 이미 정렬된 데이터에 `filter()` 함수를 적용하기 때문에, 필터링된 데이터의 순서가 보장됨
